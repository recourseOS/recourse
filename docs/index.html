The problem

Agents cause damage because they can't see consequences.

An agent reads "will be destroyed" and doesn't know if that means recoverable or gone forever. It runs the command. The database disappears. No backup. RecourseOS is the call agents make first — before they cause the damage they're getting blamed for.

How agents use it

The preflight check pattern.

Before executing any destructive action, the agent calls RecourseOS. The response tells it exactly what to do next.

1
Agent receives destructive task
"Delete the old staging database" or terraform apply with deletions
2
Agent calls RecourseOS first
recourse_evaluate_terraform or recourse_evaluate_shell — before executing anything
3
Agent routes on decision
allowExecute the action
warnSurface recovery requirements, then execute
escalateAsk human for review with missing evidence
blockRefuse to execute, explain why
4
Agent explains with evidence
"Blocked because skip_final_snapshot=true and backup_retention_period=0 — database would be unrecoverable."
Why this matters

The difference between "I deleted your database" and "I checked first."

Agents that call RecourseOS before acting can say: "I was about to delete this, but Recourse told me it's unrecoverable — skip_final_snapshot is true, no backups exist. Blocking until you approve." That's a different conversation than "I deleted your production database."

what an agent sees vs what it needs

What the agent needs
aws_db_instance.main

evidence: skip_final_snapshot=true, backup_retention_period=0
decision: block

Coverage

Rules where safety is knowable. Review where it is not.

Provider-specific handlers cover common AWS, GCP, Azure, and Azure AD resources. Unknown resources use semantic safety signals and stay conservative when evidence is thin.

AWS
Data, identity, network
RDS, DynamoDB, S3, EBS, IAM, KMS, Secrets Manager, VPC, Route53, Lambda, queues, and more.
GCP
Storage, SQL, IAM
GCS versioning, Cloud SQL protection and backups, Secret Manager, BigQuery, IAM bindings, and long-tail review paths.
Azure
Storage, database, identity
Storage soft delete, SQL family protection signals, Cosmos DB, Key Vault, role assignments, and Azure AD resources.
Unknown
Semantic classifier
Deletion protection, backup windows, versioning, retention, and force-destroy signals are evaluated without hardcoding every resource.
The contract

A schema agents can reason about reliably.

No prose, no diffs to parse. Enum decisions, typed tiers, structured evidence. The consequence report is the product — agents consume it directly.

decision
block
Unambiguous enum the agent acts on. No interpretation required.
tier
backup
Recoverability level the agent can relay: reversible, effort, backup, unrecoverable.
evidence
array
Structured fields the agent can include in its explanation to humans.
missing
review
When evidence is incomplete, escalate to human — never guess safe.
