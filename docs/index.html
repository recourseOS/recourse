Why not just Terraform plan?

Terraform shows the diff. Recourse shows the consequence.

Terraform tells you a resource will be destroyed. Recourse tells you whether that destruction is reversible, recoverable from backup, unrecoverable, or too ambiguous to treat as safe.

Point-of-no-return checks for infrastructure mutations.

Coverage

What it checks.

Known resources use deterministic provider rules. Unknown resource types use a conservative semantic classifier when enabled, so the evaluator can generalize without marking weak evidence as safe.

01 AWS data stores RDS, DynamoDB, S3, EBS snapshots, backup retention
02 AWS platform EC2, Lambda, IAM, KMS, VPC, ELB, Route53, SNS/SQS
03 GCP storage GCS buckets, objects, versioning, force destroy, bucket IAM
04 GCP databases Cloud SQL deletion protection, backups, PITR, users
05 Azure storage Storage accounts, blobs, containers, soft delete, versioning
06 Azure databases SQL/MSSQL, PostgreSQL, MySQL, MariaDB retention signals
07 Identity and secrets IAM bindings, role assignments, service account keys, credentials
08 Unknown resources semantic safety signals, BitNet-ready contract, needs-review fallback
Design principles

Checkable verdicts, not vibes.

No inference theater
It does not guess that something “looks like” production. It reads explicit values in your plan and reports the recoverability consequence.
No severity-by-vibes
Verdicts are factual: reversible, recoverable-with-effort, recoverable-from-backup, unrecoverable, or needs-review.
No moralizing your config
It does not merely say “set deletion_protection.” It tells you what would change the recoverability verdict if you did.
Local first
It runs as a CLI on your machine or in CI. Optional cloud submission never changes the local report or local exit behavior.
Honest limits

What it does not pretend to know.

Evidence-bound. If an out-of-band backup exists but is not in the plan, state, or supplied evidence, RecourseOS will not pretend it verified recovery.

Provider rules are incremental. AWS, GCP, and Azure have first-class coverage for common destructive resources, but long-tail services still need rules and fixtures.

Unknown resources stay conservative. The semantic classifier can transfer safety patterns, but weak evidence becomes needs-review.

Cloud is optional. The private control plane can ingest consequence reports later; the public CLI remains useful without signup, agents, or hosted state.
