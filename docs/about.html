Why I built this.

15 years operating mission-critical infrastructure. Now building the consequence layer AI agents need before they touch production.

I'm Jessie. I've spent 15 years operating mission-critical infrastructure across payments, regulated financial services, government systems, and evidence platforms. The kinds of environments where a wrong destructive operation against production isn't a learning experience. It's an incident that goes to the board.

For the last few years I've also been doing something else: building and operating LLM-driven systems for production operations. I've watched, up close, what changes when AI agents are the ones acting on infrastructure, and what doesn't.

What doesn't change: production data still goes away if you delete it wrong. Backups don't restore themselves. The blast radius of a destructive call is the same whether a tired engineer made it at 2am or an autonomous agent made it in nine seconds.

What changes: nobody pauses to read the plan anymore.

RecourseOS is the layer I wanted to exist before agents started touching the systems I'd spent my career protecting. It evaluates destructive infrastructure operations and tells the agent, or the human, whether the action is recoverable, before it executes. Not a guess, not a vibe. A deterministic verdict from the actual configuration of what's about to be destroyed.

It's open source because I want agents to use it. The bad rap AI agents have been getting in production isn't unfair. The incidents are real, the losses are documented. I'd rather help close that gap than write more about it.

If you're integrating an agent with infrastructure access and you want it to know what it can and can't get back, that's what this is for.

— Jessie, Founder of RecourseOS
